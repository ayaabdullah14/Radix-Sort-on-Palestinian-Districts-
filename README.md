## Project #1: Radix Sort on Palestinian Districts (Doubly Linked List)

---

## Overview

A C console application that builds a two-level hierarchy of Palestinian **districts → towns** from an input file, sorts districts alphabetically using **Radix Sort**, and sorts towns within each district by population using **Bubble Sort**. The entire structure is implemented with a **doubly linked list**.

## Input File Format (`districts.txt`)

Each line contains:
```
DistrictName | TownName | Population
```

Example:
```
Hebron | Dura | 100000
Hebron | Halhul | 40000
Jenin | Jaba | 7000
Jenin | Yabad | 31000
Gaza | Jabalia | 20000
Gaza | Beit Hanoon | 50000
```

- Separator: `|` (pipe character)
- Maximum string length: 50 characters
- Number of entries: unlimited

---

## Output File Format (`sorted_districts.txt`)

```
Gaza District, Population = 70000
Beit Hanoon, 50000
Jabalia, 20000
Hebron District, Population = 140000
Halhul, 40000
Dura, 100000
```

## Menu Options

| Option | Description |
|---|---|
| 1 | Load input file (`districts.txt`) |
| 2 | Print loaded data (unsorted, input format) |
| 3 | Sort districts alphabetically using Radix Sort |
| 4 | Sort towns within each district by population (ascending) |
| 5 | Print sorted data |
| 6 | Add a new district (with one town) |
| 7 | Add a new town to an existing district |
| 8 | Delete a town from a specific district |
| 9 | Delete a complete district (deletes all its towns first) |
| 10 | Show total Palestine population, max and min town population |

| 12 | Change the population of a specific town |
| 13 | Save sorted data to `sorted_districts.txt` |
| 14 | Exit |

## Data Structures

### Town (doubly linked list node)
```c
typedef struct Town {
    char name[50];
    int population;
    struct Town* next;
    struct Town* prev;
} Town;
```

### District (doubly linked list node, owns a Town list)
```c
typedef struct District {
    char district[50];
    int totalpopulation;
    struct Town* head;
    struct District* next;
    struct District* prev;
} District;
```

## Algorithms

### Radix Sort (districts, alphabetical)
- Sorts district names character by character from **right to left**
- Uses 64 buckets: digits (0–9), uppercase (A–Z), lowercase (a–z), other
- Time complexity: **O(k × n)** where k = max name length, n = number of districts

### Bubble Sort (towns by population)
- Applied to the town list of each district independently
- Swaps both `name` and `population` fields to keep data consistent
- Stops early if no swap occurs in a pass

## Bug Fixes Applied

The following bugs were found and fixed in the original submission:

| # | Location | Bug | Fix |

| 1 | `bubbleSort` | Crashed on districts with a single town (`current->next` was NULL) | Added null check; wrapped in do-while with early continue |
| 2 | `saveToFile` | Started from `L->next`, skipping the first district | Changed to start from `L` |
| 3 | `saveToFile` | Wrong output format (`Population =` before value) | Corrected to `District, Population = value` |
| 4 | `saveToFile` | Re-added head town population on every iteration | Removed the erroneous line |

| 6 | `DeleteList` | Had `Town*` parameter but cast internals to `District*` | Renamed to `DeleteTownList`, corrected to iterate `Town*` nodes |
| 7 | `DeleteDistrict` | `l = current->next` was local; head not updated when first district deleted | Changed to `District** l` (pointer-to-pointer) |
| 8 | `AddDistrict` | Same head-update problem as above | Changed to `District** l` |
| 9 | `printSortInfo` | Called `sortDistricts` and `bubbleSort` inside the print loop on every town | Moved sorting to caller (options 3 & 4); print loop is read-only |
